# App 审核被拒常见原因汇总

> 50+ 真实被拒案例及解决方案，持续更新

## Apple App Store

### Guideline 2.1 — App Completeness（App 完整性）
| 原因 | 解决方案 |
|---|---|
| App 崩溃 | 全面测试，使用 TestFlight 内测 |
| 功能不完整 | 移除所有「Coming Soon」占位 |
| 占位内容 | 替换为真实内容 |
| 链接失效 | 检查所有链接 |
| 登录失败 | 提供测试账号 |

### Guideline 2.3 — Inaccurate Metadata（元数据不准确）
| 原因 | 解决方案 |
|---|---|
| 截图与实际不符 | 使用真实截图 |
| 描述夸大 | 客观描述功能 |
| 关键词堆砌 | 只用相关关键词 |
| 图标与实际不符 | 使用真实 App 图标 |

### Guideline 2.5.1 — Performance: Software Requirements
| 原因 | 解决方案 |
|---|---|
| 使用私有 API | 只使用公开 API |
| WebView 包装 | 增加原生功能 |
| 读取其他 App 数据 | 使用系统提供的共享机制 |

### Guideline 4.2 — Minimum Functionality（最低功能要求）
| 原因 | 解决方案 |
|---|---|
| 功能过于简单 | 增加独特功能 |
| 只是网页打包 | 添加原生功能（推送、离线等） |
| 没有实际价值 | 明确 App 的独特价值 |

### Guideline 4.3 — Spam（垃圾应用）
| 原因 | 解决方案 |
|---|---|
| 重复应用 | 每个 App 需有独特功能 |
| 误导用户 | 明确 App 功能 |
| 低质量内容 | 提升内容质量 |

### Guideline 5.1.1 — Data Collection and Storage
| 原因 | 解决方案 |
|---|---|
| 未说明数据收集 | 完善隐私政策 |
| 未获得用户同意 | 添加同意弹窗 |
| 数据存储不安全 | 使用加密存储 |

### Guideline 5.1.2 — Data Use and Sharing
| 原因 | 解决方案 |
|---|---|
| 数据共享未告知 | App 内添加说明 |
| 第三方 SDK 未声明 | 声明所有第三方 SDK |

### ⚠️ RunOrb 实战被拒经验
1. **隐私政策 URL 不可访问**
   - 原因：SPA 路由把 `/privacy.html` 重定向到首页
   - 解决：使用独立 HTML 文件

2. **权限说明不充分**
   - 原因：蓝牙权限用途描述过于简单
   - 解决：详细说明每个权限的用途

3. **测试账号未提供**
   - 原因：审核员无法登录测试
   - 解决：在审核信息中提供测试账号

## Google Play

### 政策违规
| 原因 | 解决方案 |
|---|---|
| 隐私政策 URL 不可访问 | 确保直接返回 HTML |
| 数据安全表不准确 | 如实声明所有数据收集 |
| 权限使用不合理 | 说明每个权限用途 |
| 目标受众设置不当 | 如实设置 |
| 知识产权侵权 | 不使用他人商标/版权 |

### Metadata 违规
| 原因 | 解决方案 |
|---|---|
| 应用描述含不当内容 | 修改描述 |
| 截图不当 | 使用合规截图 |
| 图标不当 | 修改图标 |

### ⚠️ RunOrb 被拒经验
1. **隐私政策 URL 重定向**
   - 原因：CF Pages SPA 路由把 `/privacy.html` 重定向到首页
   - 解决：使用独立 HTML 文件

2. **FOREGROUND_SERVICE_CONNECTED_DEVICE 权限**
   - 原因：需上传演示视频展示功能
   - 解决：录制 YouTube 视频展示功能

3. **Closed Testing 未完成**
   - 原因：新账号必须先完成 Closed Testing
   - 解决：完成 12 测试员 × 14 天测试

## 通用建议

### 审核前自检清单
- [ ] App 不崩溃、无明显 bug
- [ ] 所有功能可用
- [ ] 隐私政策 URL 可直接访问（无重定向）
- [ ] 权限使用说明清晰
- [ ] 截图与实际功能一致
- [ ] 应用描述客观准确
- [ ] 提供测试账号（如需登录）
- [ ] 无未授权的第三方内容
- [ ] 数据安全表如实填写
- [ ] 内容分级问卷如实填写

### 被拒后应对策略
1. **仔细阅读拒绝原因** — 不要急于修改
2. **理解审核员视角** — 他们可能不完全理解你的 App
3. **针对性修改** — 只改被拒的部分，不要大改
4. **附上说明** — 重新提交时附上修改说明
5. **必要时申诉** — 如果认为审核有误，可以申诉

### 审核沟通技巧
- 礼貌、专业
- 直接回答审核员的问题
- 提供具体的修改说明
- 必要时附上截图/视频
